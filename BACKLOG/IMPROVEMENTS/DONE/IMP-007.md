# IMP-007: Supprimer oneshot, absorber stack table dans fichier partagé

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** cli
**Date creation:** 2026-03-21

---

## Description

Le module oneshot contredit les règles fondamentales de CLAUDE.md ("comprendre avant de coder") sans clause de suspension explicite. Sa stack detection table diverge de celle de `/check`. La valeur ajoutée est faible face à un supervisor en mode "fast track".

Décision : supprimer le module standalone et absorber les parties utiles.

## Approche

1. **Extraire** la stack detection table de oneshot vers un fichier partagé (ex: `shared/stack-detection.md`)
2. **Mettre à jour** `/check` pour référencer ce fichier partagé
3. **Supprimer** le répertoire `oneshot/` entièrement
4. **Retirer** oneshot de `install.sh` (MODULES array) et du README
5. **Nettoyer** les éventuels fichiers installés par oneshot (commande slash, etc.)
6. **Mettre à jour** la table des modules dans install.sh et README pour refléter les modules réels

## Fichiers concernes

- `oneshot/` — suppression complète
- `pre-commit-gate/commands/check.md` — utiliser la stack table partagée
- `install.sh` — retirer oneshot du MODULES array
- `README.md` — retirer oneshot, mettre à jour la table

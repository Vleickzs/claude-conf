# IMP-015: tab-titles — mise à jour du bloc .zshrc existant à la réinstall

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** S
**Tags:** cli, ux
**Date creation:** 2026-03-21

---

## Description

L'installeur tab-titles détecte le marker `# ── Claude Code Tab Titles` dans `.zshrc` et skip si déjà présent (ligne 106). Conséquence : les utilisateurs existants gardent l'ancienne palette (avec emojis cassés 🩷🩵⚪) après une mise à jour du module.

Découvert par le worker BUG-005 : après le fix de la palette, les users existants ne bénéficient pas du correctif sans intervention manuelle.

## Approche

Lors de l'install, si le marker existe :
1. Supprimer le bloc entre `# ── Claude Code Tab Titles` et `# ── End Claude Code Tab Titles`
2. Réinjecter le nouveau bloc

Le pattern de markers start/end est déjà en place (lignes 111 et 182).

## Fichiers concernes

- `tab-titles/install.sh` — section 4 (aliases shell)

## Criteres d'acceptation

- [ ] La réinstall met à jour le bloc .zshrc
- [ ] L'ancien bloc est proprement supprimé (pas de résidu)
- [ ] Un .zshrc sans le marker fonctionne toujours (premier install)

## Tests de validation

- [ ] Installer, vérifier le bloc. Modifier la palette manuellement. Réinstaller. Vérifier que la palette est revenue au standard.

## Description

Le module `critical-thinking` fonctionne mais n'est pas encore integre dans l'ecosysteme CLI de claude-conf. De plus, `install.sh` utilise `sed -i ''` qui est macOS-only, et il manque une logique d'update (relancer install.sh quand le snippet a change).

Deux bugs trouves en production :
1. `sed -i ''` macOS-only
2. Step 4/4 (patch supervisor local) matche `## YOUR ROLE` (anglais) mais la copie locale peut etre en francais (`## TON ROLE`) → le patch echoue silencieusement

## Taches

### 1. Portabilite sed (Linux)

`critical-thinking/install.sh` ligne 69 utilise `sed -i ''` (syntaxe macOS). Sur Linux, il faut `sed -i` sans argument vide.

**Approche recommandee** : detecter l'OS et adapter la commande sed. Pattern :
```bash
if [[ "$OSTYPE" == "darwin"* ]]; then
    sed -i '' 's/...' "$file"
else
    sed -i 's/...' "$file"
fi
```

Verifier si les autres modules (handoff-kit) ont le meme probleme et uniformiser.

### 2. Logique d'update dans install.sh (au lieu de SKIP)

Actuellement, quand les markers `<!-- critical-thinking:start -->` sont deja presents, install.sh affiche SKIP. Ca empeche de mettre a jour le contenu quand le snippet evolue.

**Nouveau comportement** :
- Si markers presents → **supprimer le contenu entre markers** (inclus) puis **reinjecter** le snippet a jour
- Afficher `UPDATED` en bleu au lieu de `SKIP`
- Meme logique pour le patch supervisor (step 4/4)

**Approche** : utiliser sed pour supprimer entre markers, puis append le nouveau contenu. Pattern :
```bash
if grep -q '<!-- critical-thinking:start -->' "$CLAUDE_MD"; then
    # Remove old content between markers
    sed_inplace '/<!-- critical-thinking:start -->/,/<!-- critical-thinking:end -->/d' "$CLAUDE_MD"
    # Append new content
    cat "$SNIPPET_FILE" >> "$CLAUDE_MD"
    echo "UPDATED"
else
    # First install
    cat "$SNIPPET_FILE" >> "$CLAUDE_MD"
    echo "OK"
fi
```

### 3. Fix bug patch supervisor — matcher FR et EN

Step 4/4 cherche `## YOUR ROLE` pour positionner l'insertion du bloc POSTURE. Mais la copie locale peut etre en francais (`## TON ROLE`).

**Approche** : matcher les deux variantes. Le while-read doit tester :
```bash
if [ "$line" = "## YOUR ROLE" ] || [ "$line" = "## TON ROLE" ]; then
```

### 4. Root install.sh — ajouter critical-thinking

Ajouter `critical-thinking` dans le tableau `MODULES` et `DESCRIPTIONS` de `install.sh` (root).

### 3. CLI bin/claude-conf — ajouter sync

Ajouter le case `critical-thinking` dans `cmd_sync()` de `bin/claude-conf` pour tracker :
- `claude-md/critical-thinking.md` → `~/.claude/CLAUDE.md` (attention : c'est un append, pas une copie directe — le sync devra comparer le contenu entre markers)

### 4. CLI bin/claude-conf — ajouter disable/enable

Verifier que `disable`/`enable` fonctionne avec critical-thinking (devrait marcher car le CLI detecte les modules par la presence de `install.sh`).

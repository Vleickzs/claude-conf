# IMP-006: supervisor — dédupliquer bloc POSTURE vs CLAUDE.md critical-thinking

**Type:** Improvement
**Statut:** A faire
**Priorite:** Basse
**Complexite:** XS
**Tags:** perf
**Date creation:** 2026-03-21

---

## Description

Au runtime, quand le supervisor est lancé sur un projet avec critical-thinking installé, Claude charge les mêmes règles anti-complaisance deux fois :
1. Via `~/.claude/CLAUDE.md` (injecté par critical-thinking/install.sh)
2. Via le bloc POSTURE dans `supervisor/commands/supervisor.md`

C'est du gaspillage de tokens et un risque de divergence si une copie est mise à jour sans l'autre.

## Approche

Supprimer le bloc POSTURE inline du supervisor et le remplacer par une référence :
```markdown
## POSTURE

Les règles anti-complaisance de critical-thinking (CLAUDE.md) s'appliquent.
En complément, le supervisor applique ces règles spécifiques :
- [règles PROPRES au supervisor uniquement, pas de duplication]
```

## Fichiers concernes

- `supervisor/commands/supervisor.md` — supprimer le bloc POSTURE dupliqué
- `supervisor/install.sh` — supprimer l'injection du bloc si applicable

## Criteres d'acceptation

- [ ] Le supervisor n'injecte plus le bloc critical-thinking en doublon
- [ ] Les règles propres au supervisor (pas dans critical-thinking) sont conservées
- [ ] Le comportement anti-complaisance est maintenu via CLAUDE.md

## Tests de validation

- [ ] Lancer /supervisor, vérifier que le comportement anti-complaisance fonctionne
- [ ] Compter les tokens du prompt supervisor — doit être réduit de ~170 lignes

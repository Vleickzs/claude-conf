# IMP-012: command-guard — wrapper shell pre-filter pour skip non-Bash

**Type:** Improvement
**Statut:** A faire

## Description

command-guard se déclenche sur CHAQUE PreToolUse, pas seulement Bash. Pour Read, Write, Edit, Grep, Glob (~70% des tool uses), Bun démarre, parse le JSON, instancie le validator, puis vérifie `toolName !== "Bash"` et exit. Tout ce travail pour rien.

Un wrapper shell ultra-léger qui extrait le `tool_name` AVANT de lancer Bun élimine ~70% des invocations Bun inutiles.

## Approche

Créer un wrapper `command-guard-wrapper.sh` :
```bash
#!/bin/bash
# Fast pre-filter: only launch Bun for Bash commands
input=$(cat)
tool_name=$(echo "$input" | jq -r '.tool_name // empty' 2>/dev/null)
if [ "$tool_name" != "Bash" ]; then exit 0; fi
echo "$input" | bun ~/.claude/scripts/command-guard/dist/cli.js
```

Le hook settings.json pointe vers le wrapper au lieu de Bun directement.

Coût du wrapper seul (jq + comparaison) : ~15-20ms vs ~150-300ms pour Bun.
Pour les 70% d'appels non-Bash, on économise ~130-280ms chacun.

## Fichiers concernes

- `command-guard/hooks/command-guard-wrapper.sh` — nouveau fichier
- `command-guard/install.sh` — installer le wrapper + pointer le hook vers lui
- settings.json hook config — pointer vers le wrapper

## Criteres d'acceptation

- [ ] Les commandes Bash sont toujours validées par command-guard
- [ ] Les outils non-Bash (Read, Write, Edit, etc.) ne lancent PAS Bun
- [ ] Le wrapper est < 10 lignes
- [ ] La sécurité est identique

## Tests de validation

- [ ] `echo '{"tool_name":"Read"}' | bash command-guard-wrapper.sh` → exit 0 instantané
- [ ] `echo '{"tool_name":"Bash","tool_input":{"command":"rm -rf /"}}' | bash command-guard-wrapper.sh` → bloqué

# IMP-011: command-guard — pre-bundle avec bun build pour eliminer le cold start

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** perf
**Date creation:** 2026-03-21

---

## Description

command-guard lance `bun cli.ts` (source TypeScript) sur chaque commande Bash. Bun doit parser et JIT-compiler le TS à chaque invocation. Coût : ~150-300ms par appel. Sur une session active (50-100 commandes Bash), c'est 15-30 secondes de latence pure cumulée.

Pre-builder le TypeScript en JS à l'install élimine le parsing/compilation à chaque appel.

## Approche

1. Ajouter dans `command-guard/install.sh` après la copie des sources :
   ```bash
   cd "$SCRIPT_DIR" && bun build src/cli.ts --outfile dist/cli.js --target bun
   ```

2. Modifier le hook dans `settings.json` pour pointer vers `dist/cli.js` au lieu de `src/cli.ts`

3. Ajouter `dist/` dans `.gitignore` de command-guard (artifact de build, pas source)

4. Les tests continuent de tourner sur les sources (`bun test` dans src/)

## Fichiers concernes

- `command-guard/install.sh` — ajouter l'étape bun build
- `command-guard/.gitignore` — ajouter dist/
- Le hook dans settings.json — pointer vers dist/cli.js

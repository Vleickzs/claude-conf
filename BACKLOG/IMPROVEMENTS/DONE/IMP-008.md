# IMP-008: handoff — rotation des backups (garder 35 derniers)

**Type:** Improvement
**Statut:** A faire
**Priorite:** Basse
**Complexite:** XS
**Tags:** cli
**Date creation:** 2026-03-21

---

## Description

Les fichiers `AUTOBACKUP-*.json` et `AUTOBACKUP-*.md` dans `~/.claude/handoff-system/sessions/` s'accumulent indéfiniment. Un utilisateur actif accumule des Go sur le long terme.

## Approche

Ajouter 2 lignes en fin de `pre-compact-handoff.sh`, après la sauvegarde :
```bash
ls -t ~/.claude/handoff-system/sessions/AUTOBACKUP-*.json 2>/dev/null | tail -n +36 | xargs rm -f
ls -t ~/.claude/handoff-system/sessions/AUTOBACKUP-*.md 2>/dev/null | tail -n +36 | xargs rm -f
```

Garde les 35 derniers de chaque type, supprime le reste.

## Fichiers concernes

- `handoff-kit/hooks/pre-compact-handoff.sh` — ajouter la rotation après le backup

## Criteres d'acceptation

- [ ] Après un backup, max 35 fichiers .json et 35 fichiers .md
- [ ] Les plus récents sont conservés
- [ ] Pas d'erreur si moins de 35 fichiers existent

## Tests de validation

- [ ] Créer 40 fichiers AUTOBACKUP-*.json factices, déclencher le hook, vérifier qu'il en reste 35

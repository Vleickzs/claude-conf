# IMP-010: worker rapport — ajouter section tests manquants (boy scout)

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** XS
**Tags:** supervisor
**Date creation:** 2026-03-21

---

## Description

Le rapport worker obligatoire n'inclut pas de section sur les tests manquants détectés pendant le travail. De plus, la boy scout rule actuelle dit "tu en écris" alors que le flow validé est : le worker DÉTECTE et RAPPORTE, le supervisor CRÉE les tickets.

Deux modifications dans `supervisor/commands/supervisor.md` :

## Modification 1 — Règle boy scout (bloc universel worker)

Remplacer la règle 11 :
```
# AVANT
11. **Boy scout rule pour les tests** — si un fichier que tu touches n'a pas de tests,
    tu en écris. La couverture progresse avec chaque ticket. Pas d'excuse.

# APRES
11. **Boy scout rule pour les tests** — si un fichier que tu touches n'a pas de tests,
    ou si tu identifies des tests manquants, NOTE-LE dans la section "Tests manquants
    (boy scout)" du rapport. NE LES ECRIS PAS toi-même — le supervisor créera les
    tickets. Ton job : détecter et rapporter, pas implémenter.
```

## Modification 2 — Section rapport worker

Ajouter dans le bloc rapport :
```markdown
### Tests manquants (boy scout)
- `path/to/file.ext` — pas de test pour [fonctionnalité]. Suggestion : [type de test]
- Ou : "Aucun test manquant détecté"
```

## Modification 3 — Section 5bis supervisor

Ajouter l'analyse des tests manquants dans la liste des verdicts :
Le supervisor crée des tickets IMP pour chaque entrée pertinente.

## Fichiers concernes

- `supervisor/commands/supervisor.md` — règle 11 du bloc universel + bloc rapport + section 5bis

# IMP-005: handoff — supprimer CRITICAL auto-execute, garder WARNING only

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** cli
**Date creation:** 2026-03-21

---

## Description

Le comportement CRITICAL (> 80% contexte) injecte `[SYSTEM-HANDOFF-CRITICAL]` et attend de Claude qu'il auto-exécute `/handoff`. C'est architecturalement cassé : Claude ne peut pas auto-invoquer un slash command.

Avec 1M tokens par conversation, le seuil de 80% (800k tokens) est rarement atteint en usage normal. Le CRITICAL automatique est à la fois cassé ET inutile.

## User Story

**En tant que** utilisateur
**Je veux** être averti quand le contexte est haut, sans interruption forcée
**Afin de** décider moi-même quand faire un handoff

## Approche

1. **Garder** le WARNING à 70% — informatif, non-intrusif
2. **Supprimer** le bloc CRITICAL dans `context-monitor.sh` (plus d'injection `[SYSTEM-HANDOFF-CRITICAL]`)
3. **Supprimer** les règles CRITICAL dans CLAUDE.md (le bloc "ARRET IMMEDIAT OBLIGATOIRE")
4. **Garder** le PreCompact backup comme filet de sécurité silencieux
5. **Garder** `/handoff` comme commande manuelle

## Fichiers concernes

- `handoff-kit/hooks/context-monitor.sh` — supprimer le bloc CRITICAL
- `~/.claude/CLAUDE.md` — supprimer les règles CRITICAL (section handoff)
- `handoff-kit/README.md` — mettre à jour la documentation

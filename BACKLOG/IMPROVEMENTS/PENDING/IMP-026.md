# IMP-026: command-guard — cleanup ancien hook lors upgrade (migration bun src → wrapper)

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** XS
**Tags:** cli
**Date creation:** 2026-03-22

---

## Description

Lors d'une réinstall de command-guard, l'ancien hook `bun $INSTALL_DIR/src/cli.ts` n'est pas supprimé de settings.json. Le nouveau hook (wrapper) est ajouté en plus. L'utilisateur se retrouve avec 2 entrées PreToolUse pour command-guard, la commande étant exécutée deux fois.

Découvert par le worker IMP-011+012.

## Approche

Dans install.sh, avant d'ajouter le nouveau hook :
1. Supprimer toute entrée PreToolUse dont la commande contient `command-guard` (ancien ou nouveau format)
2. Puis ajouter le nouveau hook proprement

## Fichiers concernes

- `command-guard/install.sh` — section jq merge

## Criteres d'acceptation

- [ ] Réinstall sur un système avec l'ancien hook → une seule entrée après install
- [ ] Premier install → fonctionne normalement

# IMP-023: supervisor — git diff scopé aux fichiers du rapport (fausse détection scope violation)

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** supervisor
**Date creation:** 2026-03-21

---

## Description

En multi-workers parallèles, le supervisor lance `git diff` global (étape 4 de la section 5 "Validate reports"), voit TOUTES les modifications du working tree (y compris celles d'autres workers), et accuse à tort son worker d'avoir débordé du scope.

Le filtrage existe déjà pour le staging (étape 5 : "stage ONLY files from the report") mais PAS pour l'évaluation. Le supervisor filtre au moment du commit mais pas au moment de la review. C'est le trou.

## Cause racine

Section 5, étape 4 dit simplement "Check the diff — git diff to see exactly what changed" sans préciser de scoper le diff.

## Approche

Modifier l'étape 4 de la section 5 dans `supervisor/commands/supervisor.md` :

### Avant :
```
4. **Check the diff** — `git diff` to see exactly what changed
```

### Après :
```
4. **Check the diff** — use `git diff -- file1 file2 file3` scoped to the files listed
   in the worker's "Files modified" section. NEVER run a global `git diff`.
   Other changes in the working tree are from other parallel workers — do NOT evaluate
   or flag them as scope violations.
   If the post-tool-use manifest exists for this worker's session_id, compare the manifest
   to the report first — the manifest is the mechanical source of truth.
```

## Fichiers concernes

- `supervisor/commands/supervisor.md` — section 5 étape 4

## Criteres d'acceptation

- [ ] L'étape 4 dit explicitement de scoper le diff
- [ ] L'instruction mentionne de NE PAS évaluer les fichiers d'autres workers
- [ ] Le manifest est mentionné comme source de vérité alternative

## Tests de validation

- [ ] Lancer /supervisor sur un projet avec des modifications d'un autre workflow dans le working tree
- [ ] Vérifier que le supervisor ne flag pas les fichiers hors scope du worker

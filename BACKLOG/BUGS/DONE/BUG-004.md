# BUG-004: install.sh — footgun `((count++))` avec `set -e`

**Type:** Bug
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** cli
**Date creation:** 2026-03-21

---

## Description

Dans `install.sh`, les compteurs `success` et `failed` sont incrémentés avec `((var++))`. En bash avec `set -e`, si la variable vaut 0, `((0++))` évalue l'ancienne valeur (0), ce qui donne un exit code 1 (0 = faux en arithmétique bash). `set -e` tue le script.

Scénario concret : si le premier module échoue, `((failed++))` avec `failed=0` → exit code 1 → script mort.

## Fichiers concernes

- `install.sh` — toutes les occurrences de `((success++))` et `((failed++))`

## Approche

Remplacer toutes les occurrences :
```bash
# AVANT
((success++))
((failed++))

# APRES
success=$((success + 1))
failed=$((failed + 1))
```

Scanner aussi les autres scripts du projet pour le même pattern.

- [ ] Le résumé final affiche les bons compteurs

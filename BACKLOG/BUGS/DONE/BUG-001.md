# BUG-001: Tab-titles — worker sessions affichent SUP au lieu de WORK (vert)

**Type:** Bug
**Statut:** Fait
**Priorite:** Haute
**Complexite:** XS
**Scope:** both
**Tags:** tab-titles, hooks
**Date creation:** 2026-03-20

---

## Description

Les sessions worker lancees avec `ccd` + prompt colle affichent "🔴 SUP" au lieu de "🟢 FEAT-001" (vert). Le titre d'onglet ne detecte pas correctement le mode worker quand le prompt contient des chemins de fichiers incluant `/supervisor`.

## Cause racine

`tab-titles/hooks/tab-title.sh` ligne 39 :
```bash
if echo "$PROMPT" | grep -qi '/supervisor'; then
```

Le pattern `/supervisor` matche dans les chemins de fichiers comme `supervisor/commands/supervisor.md`. Or les prompts worker contiennent souvent ces paths (references aux fichiers du module supervisor). Le grep trouve `/supervisor` dans le path → etat force a `SUP` → la detection ticket (ligne 41-51) est bypassee car guard `$CURRENT_STATE != "SUP"`.

## Fix

Remplacer ligne 39 :
```bash
if echo "$PROMPT" | grep -qiE '(^|[[:space:]])/supervisor'; then
```

Ca matche `/supervisor` uniquement en debut de ligne ou apres un espace — plus dans les chemins de fichiers.

## Criteres d'acceptation

- [ ] Un prompt worker contenant `supervisor/commands/supervisor.md` ne trigger pas le mode SUP
- [ ] La commande `/supervisor` (en debut de message ou apres espace) trigger toujours le mode SUP
- [ ] Un worker colle avec un ticket FEAT-XXX affiche bien "🟢 FEAT-XXX" en vert

## Fichiers concernes

- `tab-titles/hooks/tab-title.sh` (ligne 39 — 1 ligne a modifier)

## Tests de validation

- [ ] Lancer `ccd`, coller un prompt contenant "supervisor/commands/supervisor.md" et "FEAT-001" → onglet vert FEAT-001
- [ ] Lancer `cc`, taper `/supervisor` → onglet rouge SUP
- [ ] Lancer `cc`, taper "lis supervisor/commands/supervisor.md" → onglet NE passe PAS en SUP

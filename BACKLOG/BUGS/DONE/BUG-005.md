# BUG-005: tab-titles — couleurs par projet toujours blanches

**Type:** Bug
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** ui
**Date creation:** 2026-03-21

---

## Description

Les titres d'onglets affichent le bon mode (SUP, WORK, etc.) mais la couleur du dot est toujours blanche quel que soit le projet. La couleur devrait être unique par projet (hash du nom via `cksum`).

Deux problèmes identifiés :
1. **Off-by-one** : la fonction shell `.zshrc` utilise `% 8 + 1` (array zsh 1-indexed) tandis que le hook `tab-title.sh` utilise `% 8` (array bash 0-indexed). Les deux ne pointent pas la même couleur.
2. **Possiblement** : le `cksum` ne reçoit pas le bon input (nom de projet vide ou path au lieu du nom).

## Fichiers concernes

- `tab-titles/install.sh` — fonction `_cc_dot` injectée dans `.zshrc` (~ligne 119)
- `tab-titles/hooks/tab-title.sh` — calcul d'index couleur (~ligne 27)

## Approche

1. Lire les deux fichiers et comparer les palettes de couleurs ET le calcul d'index
2. Identifier pourquoi le résultat est toujours blanc (même index ? palette cassée ? input vide ?)
3. Unifier le calcul : même palette, même formule, même input
4. Tester avec 3-4 noms de projet différents pour vérifier des couleurs différentes

## Criteres d'acceptation

- [ ] Deux projets différents affichent des couleurs de dot différentes
- [ ] La couleur est consistante entre le lancement shell et les hooks mid-session
- [ ] La palette utilise au moins 6 couleurs distinctes visuellement

## Tests de validation

- [ ] Ouvrir 3 onglets sur 3 projets différents
- [ ] Vérifier visuellement que les dots sont de couleurs différentes
- [ ] Relancer une session sur le même projet — vérifier que la couleur est stable

# BUG-003: command-guard — `"deny"` au lieu de `"block"` dans HookOutput

**Type:** Bug
**Statut:** A faire
**Priorite:** Critique
**Complexite:** XS
**Tags:** security
**Date creation:** 2026-03-21

---

## Description

Dans `command-guard/src/cli.ts` ligne 105, le `permissionDecision` est mis à `"deny"` quand une commande dangereuse est détectée. Or le protocole hooks Claude Code attend la valeur `"block"` pour bloquer réellement l'exécution.

Conséquence : les commandes dangereuses détectées ne sont potentiellement PAS bloquées — la valeur sécurité entière du module est compromise.

## Fichiers concernes

- `command-guard/src/cli.ts` — ligne 105, changer `"deny"` → `"block"`
- `command-guard/src/lib/types.ts` — vérifier que le type `HookOutput.permissionDecision` inclut `"block"`

## Approche

1. Dans `types.ts`, vérifier/corriger le type : `"allow" | "block" | "ask"`
2. Dans `cli.ts`, remplacer `"deny"` par `"block"`
3. Vérifier qu'il n'y a pas d'autres occurrences de `"deny"` dans le code
4. Lancer les tests existants

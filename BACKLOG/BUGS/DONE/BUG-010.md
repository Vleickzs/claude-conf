# BUG-010 — tab-titles: titre écrasé au démarrage de Claude Code (🟣 au lieu du bon emoji)

**Priorité:** Haute
**Module:** tab-titles

## Symptôme

Au lancement de `cc`/`ccd`/`ccs`/`ccw`, le titre d'onglet Terminal.app affiche 🟣 (ou un emoji incorrect) pendant le démarrage de Claude Code, quel que soit le projet. Le bon titre n'apparaît qu'au premier message (quand le hook `UserPromptSubmit` tire).

## Cause racine

L'initialisation de Claude Code (Ink framework) écrit des séquences CSI au terminal (alt screen, cursor visibility, focus events) qui provoquent un reset/flash du titre OSC 1 dans Terminal.app. Le flag `CLAUDE_CODE_DISABLE_TERMINAL_TITLE=1` empêche Claude Code de setter son propre titre, mais n'empêche pas l'effet de bord de l'initialisation terminal sur le titre existant.

Le titre défini par les fonctions zsh (`_cc_set_tab`) AVANT le lancement est correct mais se fait écraser pendant le bootstrap d'Ink.

## Solution

Utiliser le hook `SessionStart` de Claude Code (disponible v2.1.81+) pour ré-appliquer le titre APRÈS l'initialisation complète.

### Approche technique

1. **Fonctions zsh (cc, ccd, ccs, ccw):** Exporter `CC_TAB_TITLE` et `CC_WIN_TITLE` comme env vars (héritées par Claude Code et ses hooks)
2. **Nouveau hook `session-tab-title.sh`:** Hook `SessionStart` qui lit les env vars et applique OSC 1/2 via `/dev/tty`
3. **install.sh:** Copier le nouveau hook + ajouter l'entrée SessionStart dans settings.json
4. **supervisor/commands/supervisor.md:** Ajouter export des env vars dans le template de launch script worker

### Contrainte

PAS de background process qui écrit dans le tty.

## Fichiers impactés

- `tab-titles/hooks/session-tab-title.sh` — NOUVEAU
- `tab-titles/install.sh` — ajout env vars exports + setup SessionStart hook
- `supervisor/commands/supervisor.md` — ajout env vars dans template launch script

1. `cd ~/Dev/claude-conf && ccd` → vérifier que le titre est ⚙️ CC dès le démarrage

3. `ccs` → vérifier 🔴 SUP immédiat
4. Taper un message → vérifier que le titre ne change pas (reste correct)

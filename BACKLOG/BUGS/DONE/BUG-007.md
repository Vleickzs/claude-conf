# BUG-007: command-guard — regex `rm -rf` ne matche pas en fin de string

**Type:** Bug
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** security
**Date creation:** 2026-03-21

---

## Description

Dans `command-guard/src/lib/validator.ts` (lignes 56-64), les patterns de détection de `rm -rf` exigent un `\s` (whitespace) après les flags. Conséquence : `rm -rf` en fin de string (sans argument) n'est pas détecté.

Bien que `rm -rf` seul sans argument soit inoffensif, c'est un trou dans la détection qui pourrait masquer des variantes non prévues.

## Fichiers concernes

- `command-guard/src/lib/validator.ts` — les 3 regex de `containsRmRf`
- `command-guard/src/__tests__/validator.test.ts` — ajouter cas de test

## Approche

Remplacer `\s` final par `(\s|$)` dans chaque pattern :
```typescript
// AVANT
/\brm\s+(-[a-zA-Z]*r[a-zA-Z]*f[a-zA-Z]*)\s/i

// APRES
/\brm\s+(-[a-zA-Z]*r[a-zA-Z]*f[a-zA-Z]*)(\s|$)/i
```

Appliquer à toutes les variantes. Ajouter des tests pour les cas limites (fin de string, tab au lieu d'espace).

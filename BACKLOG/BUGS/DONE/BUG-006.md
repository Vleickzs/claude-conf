# BUG-006: tab-titles — `sed -i ''` casse Linux

**Type:** Bug

**Date creation:** 2026-03-21

## Description

Dans `tab-titles/install.sh` (~ligne 191), la commande `sed -i ''` utilise la syntaxe BSD (macOS). Sur Linux, GNU sed attend `sed -i` sans argument vide, ce qui fait échouer l'install silencieusement.

## Fichiers concernes

- `tab-titles/install.sh` — toutes les occurrences de `sed -i ''`
- Scanner les autres modules pour le même pattern

## Approche

Ajouter une détection OS et brancher :
```bash
if [[ "$(uname)" == "Darwin" ]]; then
    sed -i '' 's/pattern/replacement/' file
else
    sed -i 's/pattern/replacement/' file
fi
```

Ou extraire une fonction helper `_sed_inplace()` réutilisable si plusieurs occurrences.
